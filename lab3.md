# Звіт до лабораторної роботи №3а

## 1. Послідовна реалізація

У проєкті реалізовано послідовний алгоритм генерації плану харчування.

Основна логіка знаходиться у класі `MealDistributionService`, який виконує:

* розподіл добових макронутрієнтів між прийомами їжі;
* генерацію плану харчування;
* підбір продуктів для кожного прийому їжі.

У класі `MainViewModel` послідовно виконуються такі етапи:

1. Розрахунок базового обміну речовин (BMR).
2. Розрахунок добової потреби в калоріях (TDEE).
3. Розрахунок цільових значень білків, жирів та вуглеводів.
4. Генерація плану харчування.

---

## 2. Паралельна реалізація

Для прискорення генерації раціону реалізовано паралельну версію алгоритму у класі `ParallelMealGenerationService`.

Використані засоби багатопоточності платформи .NET:

* `Parallel.For`;
* `ThreadLocal<T>`;
* налаштування рівня паралелізму через параметр `degreeOfParallelism`.

Паралельно виконується генерація окремих прийомів їжі та підбір продуктів для них. Кожен потік працює незалежно з власними локальними даними, що дозволяє уникнути конфліктів доступу до спільних ресурсів.

---

## 3. Об’єктно-орієнтоване проєктування

Проєкт побудований відповідно до принципів об’єктно-орієнтованого програмування.

Використано:

* інкапсуляцію даних у моделях та сервісах;
* інтерфейси для абстрагування реалізацій;
* розділення відповідальностей між компонентами;
* архітектурний шаблон MVVM.

Основні інтерфейси:

* `IProductRepository`;
* `IMealGenerationStrategy`.

---

## 4. Використані патерни проєктування

### Repository

Використовується для роботи з локальною базою продуктів.

* `IProductRepository`
* `JsonProductRepository`

### Strategy

Дозволяє змінювати алгоритм підбору продуктів без зміни основної логіки програми.

* `IMealGenerationStrategy`
* `GreedyMealGenerationStrategy`
* `BalancedMealGenerationStrategy`

### Service / Facade

Сервіси інкапсулюють бізнес-логіку генерації раціону.

* `MealDistributionService`
* `ParallelMealGenerationService`

### MVVM

Для побудови інтерфейсу використано архітектурний шаблон MVVM.

* `MainViewModel`
* `MainWindow.xaml`
* `RelayCommand`

---

## 5. Модульне тестування

Для перевірки працездатності реалізовано набір модульних тестів.

Перевіряються:

* коректність завантаження продуктів із JSON-файлу;
* правильність розрахунку макронутрієнтів;
* відповідність результатів послідовної та паралельної реалізації;
* обробка некоректних вхідних даних.

Тести розміщені у проєкті `NutriPlan.Tests`.

---

## 6. Вимірювання продуктивності

Для порівняння продуктивності реалізовано окремий механізм бенчмарків.

Основні компоненти:

* `BenchmarkRunner`;
* `BenchmarkSuite`.

Виконується вимірювання:

* середнього часу виконання алгоритму;
* впливу кількості потоків на швидкодію;
* впливу кількості прийомів їжі на час генерації.

Результати автоматично зберігаються у CSV-файли для подальшого аналізу.
