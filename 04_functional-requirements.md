# Функціональні вимоги (Functional Requirements)

У цьому розділі описані основні функціональні можливості системи для розрахунку КБЖВ та генерації раціону харчування.

# Функціональні вимоги

## FR1 — Введення персональних даних користувача

### Опис

Користувач вводить або обирає персональні дані, необхідні для виконання розрахунків:

* стать;
* вага (кг);
* зріст (см);
* вік (роки);
* рівень фізичної активності.

### Елементи інтерфейсу

| Поле              | Прив'язка                        |
| ----------------- | -------------------------------- |
| Стать             | `MainViewModel.SelectedGender`   |
| Вага              | `MainViewModel.Weight`           |
| Зріст             | `MainViewModel.Height`           |
| Вік               | `MainViewModel.Age`              |
| Рівень активності | `MainViewModel.SelectedActivity` |

### Правила валідації

#### Вага

* числове значення;
* більше 0;
* допустимий діапазон: **20–300 кг**.

#### Зріст

* числове значення;
* більше 0;
* допустимий діапазон: **40–250 см**.

#### Вік

* ціле число;
* більше 0;
* допустимий діапазон: **10–120 років**.

#### Стать та рівень активності

* обов’язкові для вибору.

### Поведінка системи

* Некоректні дані блокують виконання розрахунку.
* Біля відповідного поля відображається повідомлення про помилку.
* Повідомлення повинні бути конкретними.

Приклад:

```text
Вага повинна бути в межах від 20 до 300 кг.
```

* Коректно введені значення зберігаються протягом поточної сесії у властивостях `MainViewModel`.

### Критерії приймання

* [ ] Усі п’ять полів присутні та доступні для редагування.
* [ ] Введення некоректних значень показує повідомлення про помилку та блокує кнопку «Розрахувати план».
* [ ] Після виправлення значення помилка зникає, а можливість виконання розрахунку відновлюється.

## FR2 — Вибір цілі користувача

### Опис

Користувач обирає одну з цілей харчування:

* Схуднення (`WeightLoss`);
* Підтримка ваги (`Maintenance`);
* Набір м’язової маси (`MuscleGain`).

### Елементи інтерфейсу

* Елемент вибору типу `ComboBox`.
* Прив’язка до:

```csharp
MainViewModel.SelectedGoal
```

* Поруч відображається коротке пояснення щодо впливу вибраної цілі:

  * дефіцит калорій;
  * підтримка поточної ваги;
  * профіцит калорій.

### Поведінка системи

Обрана ціль використовується під час розрахунку добової калорійності та макронутрієнтів.

### Критерії приймання

* [ ] Користувач може обрати лише одну ціль.
* [ ] Зміна цілі впливає на результати наступного розрахунку.

---

## FR3 — Розрахунок добової калорійності та БЖВ

### Опис

Система виконує:

1. Розрахунок базового обміну речовин (BMR) за формулою Mifflin–St Jeor.
2. Розрахунок добових енерговитрат (TDEE) із врахуванням рівня активності.
3. Коригування результату відповідно до обраної цілі.
4. Розрахунок білків, жирів та вуглеводів у грамах.

### Запуск розрахунку

За замовчуванням:

* натискання кнопки **«Розрахувати план»**;
* виклик:

```csharp
GenerateCommand → MainViewModel.Generate()
```

Додатково (необов'язково):

* автоматичний перерахунок після зміни даних користувача;
* затримка оновлення 300–500 мс (debounce).

### Вимоги до продуктивності

Розрахунок повинен виконуватися менш ніж за:

```text
200 мс
```

на типовому комп’ютері розробника.

### Відображення результатів

У панелі підсумків відображаються:

* `MacroResult.TotalCalories`
* `MacroResult.ProteinGrams`
* `MacroResult.FatGrams`
* `MacroResult.CarbsGrams`

### Критерії приймання

* [ ] Натискання кнопки «Розрахувати план» виконує розрахунок.
* [ ] Результати відповідають реалізації сервісів `CalorieCalculatorService` та `MacroCalculatorService`.
* [ ] Розрахунок відбувається без помітної затримки.
* [ ] За ввімкненого автоматичного режиму результати оновлюються після зміни вхідних даних.

## FR4 — Відображення добових цільових показників

### Опис

Система повинна відображати результати розрахунків у зрозумілому та зручному форматі.

### Вимоги до інтерфейсу

* Добова калорійність відображається як основний показник.
* Білки, жири та вуглеводи показуються в грамах.
* Формат чисел:

```text
F1
```

(один знак після коми).

* Відображається пояснення розрахунку:

```csharp
MacroResult.FormulaExplanation
```

### Критерії приймання

* [ ] Калорії та БЖВ відображаються одразу після розрахунку.
* [ ] Форматування чисел є однаковим та зрозумілим.
* [ ] Відображається пояснення, що відповідає введеним параметрам та вибраній цілі.

---

## FR5 — Генерація денного плану харчування

### Опис

На вимогу користувача система формує денний раціон харчування, що відповідає розрахованим показникам БЖВ та калорійності.

### Запуск

Генерація виконується після натискання кнопки:

```text
Розрахувати план
```

### Поведінка системи

Система:

1. Розподіляє цільові показники між прийомами їжі.
2. Формує від 2 до 5 прийомів їжі.
3. Підбирає продукти для кожного прийому.

### Індикатор виконання

Якщо генерація триває понад 200 мс:

* відображається індикатор завантаження;
* може використовуватися спінер або прогрес-бар.

Для паралельної генерації індикатор відображається до завершення процесу.

### Обробка помилок

Якщо файл продуктів відсутній або порожній, користувач бачить повідомлення:

```text
Продукти відсутні. Додайте файл Data/products.json або перевірте його налаштування.
```

Програма не повинна аварійно завершувати роботу.

### Відображення результатів

#### Список прийомів їжі

Прив’язка:

```csharp
Meals
```

Для кожного прийому відображаються:

* назва;
* калорійність;
* білки;
* жири;
* вуглеводи.

#### Список продуктів

Прив’язка:

```csharp
MealProducts
```

Для кожного прийому їжі відображаються:

* назва продукту;
* калорійність продукту.

### Критерії приймання

* [ ] Натискання кнопки «Розрахувати план» формує список прийомів їжі та продуктів.
* [ ] Під час тривалої генерації відображається індикатор виконання.
* [ ] За відсутності продуктів показується повідомлення про помилку без аварійного завершення роботи програми.

- [ ] Раціон складається з 3-х прийомів їжі (Сніданок, Обід, Вечеря).
- [ ] Сумарна калорійність та БЖВ згенерованого раціону відповідає розрахованій добовій нормі з похибкою не більше ±10%.
- [ ] Згенерований список страв відображається на екрані одразу після завершення процесу.
