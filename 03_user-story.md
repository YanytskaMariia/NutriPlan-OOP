# User Stories

## US-1 — Введення персональних параметрів

### Опис

**Як користувач (Guest)**

Я хочу вводити в інтерфейсі:

* стать;
* вагу (кг);
* зріст (см);
* вік (роки);
* рівень фізичної активності,

щоб застосунок міг використовувати ці дані для розрахунку добової калорійності та макронутрієнтів.

### Критерії приймання

* [ ] Поля інтерфейсу прив'язані до властивостей:

  * `SelectedGender`
  * `Weight`
  * `Height`
  * `Age`
  * `SelectedActivity`
* [ ] Валідація не дозволяє вводити некоректні значення (наприклад, вага ≤ 0) та блокує виконання розрахунку.
* [ ] Введені значення зберігаються у `MainViewModel` до моменту їх зміни або закриття програми.

## US-2 — Вибір цілі

### Опис

**Як користувач (Guest)**

Я хочу обрати одну ціль:

* Схуднення (`WeightLoss`);
* Підтримка ваги (`Maintenance`);
* Набір м'язової маси (`MuscleGain`),

щоб застосунок використовував правильне коригування калорійності та розподіл БЖВ.

### Критерії приймання

* [ ] Елемент вибору підтримує вибір лише однієї цілі та прив'язаний до `MainViewModel.SelectedGoal`.
* [ ] Обрана ціль передається до сервісів:

  * `CalorieCalculatorService.CalculateTargetCalories`
  * `MacroCalculatorService`
* [ ] Після зміни цілі наступний розрахунок використовує нові параметри.

---

## US-3 — Перегляд результатів розрахунку БЖВ

### Опис

**Як користувач (Guest)**

Я хочу бачити добову калорійність та кількість білків, жирів і вуглеводів після виконання розрахунку,

щоб отримати зрозумілі числові орієнтири для свого раціону.

### Критерії приймання

* [ ] У панелі підсумків відображаються:

  * `MacroResult.TotalCalories`
  * `MacroResult.ProteinGrams`
  * `MacroResult.FatGrams`
  * `MacroResult.CarbsGrams`
* [ ] Дані прив'язані до `MainViewModel.MacroResult`.
* [ ] Формат відображення відповідає інтерфейсу:

  * калорії — у ккал;
  * білки, жири та вуглеводи — у грамах з одним знаком після коми.
* [ ] Відображається текст пояснення розрахунку:

  * `MacroResult.FormulaExplanation`.

## US-4 — Генерація денного плану харчування

### Опис

**Як користувач (Guest)**

Я хочу натиснути одну кнопку («Розрахувати план»),

щоб система автоматично розрахувала БЖВ та сформувала денний план харчування відповідно до отриманих показників.

### Критерії приймання

* [ ] Команда `GenerateCommand` у `MainViewModel` виконує:

  * розрахунок калорійності через `CalorieCalculatorService`;
  * розрахунок БЖВ через `MacroCalculatorService`;
  * створення плану харчування через `MealDistributionService.GenerateMealPlan`.
* [ ] Користувач може обрати кількість прийомів їжі від 2 до 5.
* [ ] Згенерований план використовує вибрану кількість прийомів їжі.
* [ ] Якщо генерація триває понад 200 мс, інтерфейс відображає індикатор виконання.

---

## US-5 — Перегляд складу кожного прийому їжі

### Опис

**Як користувач (Guest)**

Я хочу переглядати для кожного прийому їжі його цільові макронутрієнти та список вибраних продуктів,

щоб розуміти, як сформований раціон досягає добових показників калорійності та БЖВ.

### Критерії приймання

* [ ] Список прийомів їжі (`MainViewModel.Meals`) відображає:

  * назву прийому їжі;
  * цільову калорійність;
  * кількість білків;
  * кількість жирів;
  * кількість вуглеводів.
* [ ] Колекція `MainViewModel.MealProducts` відображає для кожного прийому їжі:

  * назви продуктів;
  * калорійність кожного продукту.
* [ ] Якщо база продуктів відсутня або порожня, користувач бачить повідомлення про помилку, а застосунок продовжує працювати без аварійного завершення.
